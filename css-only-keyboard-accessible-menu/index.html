CSS-Only (Almost) Keyboard Accessible Popup Menu

This demo shows how you can support keyboard users and improve accessibility with (almost) no JavaScript:

  • The menu below has popups that are accessible from mouse from hover states as well as with the keyboard with focus states.
  • Use the :focus-within pseudo-class with a nested focusable element to open the menu. Use this instead of :focus as it won't let the user tab into the menu itself. If you only use :focus on the main menu item as soon as the user hits tab to go into the submenu this will have the effect of closing it.
  • Add either an anchor with tabindex="0" or a button to each top-level menu item allows you to capture a focus event.
  • The small onclick handler is used to force a focus for UI tools such as Vimium that simulate clicks.
